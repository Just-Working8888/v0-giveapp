# ТЕХНИЧЕСКОЕ ЗАДАНИЕ
## Мобильная платформа для благотворительности "Благотворительность"

---

## 1. ОБЩАЯ ИНФОРМАЦИЯ

### 1.1 Назначение проекта
Разработка мобильной веб-приложения для управления благотворительными пожертвованиями. Платформа позволяет донорам поддерживать благотворительные организации и следить за влиянием своих пожертвований, а благотворительным организациям управлять кампаниями сбора средств и взаимодействовать с донорами.

### 1.2 Основная цель
Создать доступную и удобную платформу, которая:
- Облегчает процесс пожертвований для отдельных граждан
- Помогает благотворительным организациям управлять сборами средств
- Обеспечивает прозрачность и отчётность о расходовании средств
- Мотивирует доноров через систему рангов и видимого влияния

### 1.3 Целевая аудитория
- **Доноры**: граждане, желающие поддержать благотворительные организации
- **Организации**: зарегистрированные благотворительные фонды и НКО

### 1.4 Язык интерфейса
Русский язык

## 2. ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

### 2.1 Модуль для донора

#### 2.1.1 Главная страница (Домашняя страница)
**Путь:** `/`

**Функциональность:**
- Отображение списка всех благотворительных организаций
- Возможность поиска организации по названию
- Фильтрация организаций по категориям:
  - Дети
  - Животные
  - Экология
  - Пожилые
  - И другие категории
- Отображение карточки организации с информацией:
  - Название и описание
  - Изображение
  - Сумма собранных средств
  - Целевая сумма
  - Прогресс сборки (в процентах)
  - Количество донаторов
  - Статус верификации (значок проверки)

**Компоненты:**
- `OrganizationCard` - карточка организации
- `CategoryFilter` - фильтр по категориям
- `BottomNav` - нижняя навигация

#### 2.1.2 История пожертвований
**Путь:** `/donations`

**Функциональность:**
- Отображение полной истории всех пожертвований донора
- Информация о каждом пожертвовании:
  - Название организации
  - Сумма
  - Дата
  - Статус (завершено/в процессе)
  - Категория
- Сумма всех пожертвований
- Фильтрация по датам или статусу

**Компоненты:**
- `DonationHistoryCard` - карточка пожертвования в истории

#### 2.1.3 Страница статистики и влияния
**Путь:** `/stats`

**Функциональность:**
- Отображение общей статистики донора:
  - Общая сумма пожертвований
  - Количество поддерживаемых организаций
  - Статус/ранг донора (системы рангов)
  - Баллы влияния
- График месячных пожертвований
- Карточки влияния с показателями:
  - Количество профинансированных операций
  - Количество спасённых животных
  - Количество посаженных деревьев
  - И другие метрики влияния
- Система рангов (например, "Золотой донор")

**Компоненты:**
- `ImpactCard` - карточка показателя влияния
- `BottomNav` - нижняя навигация

#### 2.1.4 Профиль донора
**Путь:** `/profile`

**Функциональность:**
- Просмотр и редактирование профиля:
  - Имя
  - Email
  - Телефон
  - Аватар
- Параметры уведомлений
- История активности
- Выход из учётной записи

**Компоненты:**
- `EditProfileModal` - модальное окно редактирования профиля

#### 2.1.5 Страница организации (Детали)
**Путь:** `/organization/[id]`

**Функциональность:**
- Полная информация об организации:
  - Логотип и описание
  - Статус верификации
  - Общая сумма собранных средств
  - Количество донаторов
  - История и миссия (если доступно)
- Отображение кампаний сборки средств
- Активность организации
- Возможность пожертвования

### 2.2 Модуль для организации

#### 2.2.1 Панель управления организации
**Путь:** `/org`

**Функциональность:**
- Общая статистика:
  - Название организации
  - Статус верификации
  - Общая сумма собранных средств
  - Количество донаторов
  - Количество активных кампаний
- Список активных кампаний сборки средств с показателями:
  - Название кампании
  - Целевая сумма
  - Собранная сумма
  - Количество донаторов
  - Дни до окончания
  - Статус
- Лента активности (недавние пожертвования и комментарии)
- Кнопка для создания новой кампании

**Компоненты:**
- `CampaignCard` - карточка кампании
- `BottomNavOrg` - нижняя навигация для организаций

#### 2.2.2 Аналитика организации
**Путь:** `/org/analytics`

**Функциональность:**
- Детальная аналитика по кампаниям:
  - Графики сборки средств по времени
  - Распределение донаторов по суммам
  - Источники трафика
- Статистика по демографии донаторов
- Графики тенденций

#### 2.2.3 Управление кампаниями
**Путь:** `/org/campaign/[id]`

**Функциональность:**
- Редактирование кампании:
  - Название
  - Описание
  - Целевая сумма
  - Изображение
  - Дата окончания
- Просмотр детального прогресса
- История пожертвований по кампании
- Отправка обновлений донаторам

#### 2.2.4 Отчёты организации
**Путь:** `/org/reports`

**Функциональность:**
- Создание отчётов о расходовании средств
- Загрузка документов и подтверждающих материалов
- История отчётов с датами
- Просмотр отчётов в виде карточек

**Компоненты:**
- `OrgReportCard` - карточка отчёта организации
- `ReportCard` - карточка отчёта

#### 2.2.5 Настройки организации
**Путь:** `/org/settings`

**Функциональность:**
- Редактирование профиля организации:
  - Название
  - Email
  - Номер телефона
  - Логотип
- Переключатели для параметров:
  - Принимать пожертвования
  - Публичный профиль
- Управление пользователями (члены команды)
- Безопасность и подтверждение личности
- Способы получения средств (банковские реквизиты и т.д.)

**Компоненты:**
- `EditOrgModal` - модальное окно редактирования информации организации

## 3. АРХИТЕКТУРА И ТЕХНИЧЕСКИЕ ХАРАКТЕРИСТИКИ

### 3.1 Стек технологий

**Frontend:**
- **Framework**: Next.js 16.0.10 (React)
- **Язык**: TypeScript
- **Стилизация**: Tailwind CSS + PostCSS
- **Компоненты UI**: Radix UI (полный набор примитивов)
- **Иконки**: Lucide React
- **Управление формами**: React Hook Form + @hookform/resolvers
- **Уведомления**: Sonner (toast уведомления)
- **Валидация**: Zod (если используется)
- **Тематика**: next-themes (поддержка light/dark режима)
- **Карусели**: Embla Carousel React
- **Analytics**: Vercel Analytics

**Backend:**
- Требуется реализация REST API или GraphQL API

**Хранилище данных:**
- Требуется база данных (рекомендуется PostgreSQL или MongoDB)

### 3.2 Поддерживаемые платформы
- Мобильные браузеры (iOS Safari, Chrome Mobile)
- Десктопные браузеры (Chrome, Firefox, Safari, Edge)
- Адаптивный дизайн (mobile-first)

### 3.3 Шрифты
- Google Font: Nunito (weight: 400, 600, 700, 800)
- Кириллица поддерживается

### 3.4 Оформление
- **Тема**: Light/Dark режимы
- **Цвета**: Система цветов на основе CSS переменных
- **Радиусы**: Скруглённые углы (10-20px в большинстве элементов)
- **Теневой контур**: Мягкие тени для глубины

## 8. СЦЕНАРИИ ИСПОЛЬЗОВАНИЯ

### 8.1 Сценарий донора

1. **Регистрация и вход**
   - Новый пользователь регистрируется через email/социальные сети
   - Подтверждает email
   - Заполняет профиль

2. **Поиск организации**
   - Просматривает список организаций на главной странице
   - Использует поиск для быстрого поиска
   - Фильтрует по категориям интереса

3. **Пожертвование**
   - Нажимает кнопку пожертвования на карточке организации
   - Выбирает сумму
   - Выбирает способ оплаты
   - Подтверждает платёж

4. **Отслеживание влияния**
   - Просматривает историю пожертвований
   - Видит статистику своего влияния
   - Получает уведомления об обновлениях от организаций

### 8.2 Сценарий организации

1. **Регистрация и верификация**
   - Организация регистрируется
   - Загружает документы верификации
   - Ждёт одобрения администратора

2. **Создание кампании**
   - Создаёт новую кампанию сборки средств
   - Устанавливает целевую сумму и сроки
   - Загружает описание и изображения

3. **Управление сборкой**
   - Просматривает прогресс сборки
   - Видит список донаторов
   - Отправляет обновления о кампании
   - Публикует отчёты о расходовании

4. **Аналитика**
   - Просматривает графики сборки
   - Анализирует данные донаторов
   - Отслеживает эффективность кампаний
